# TASK-Remind セットアップ手順

> チェックボックスをクリックして進捗を管理してください。

---

## 全体の流れ

```
GitHub（コード管理）
    ↓ 自動デプロイ
Render（サーバー・無料）← UptimeRobot（5分おきにping・無料）
    ↓ DB接続
Supabase（PostgreSQL・無料）
    ↑ Webhook
LINE Messaging API（通知・無料）
```

---

## STEP 1｜GitHub にアップロード ✅

- [x] [github.com](https://github.com) でアカウント作成（既存なら不要）
- [x] 「New repository」をクリック
  - Repository name: `task-remind`
  - Public / Private どちらでもOK
  - → 「Create repository」
- [x] `TASK-Remind` フォルダでターミナルを開き、以下を順番に実行

```bash
git init
git add .
git commit -m "first commit"
git branch -M main
git remote add origin https://github.com/ishi-kosei/task-remind.git
git push -u origin main
```

- [x] GitHubのリポジトリページを開き、ファイルが表示されていることを確認
  - → https://github.com/ishi-kosei/task-remind

---

## STEP 2｜Supabase でデータベース作成 ✅

- [x] [supabase.com](https://supabase.com) → 「Start your project」でアカウント作成
- [x] 「New project」をクリック
  - Name: `task-remind`
  - Region: Northeast Asia (Tokyo)
- [x] 左メニュー「**Settings**」→「**Database**」→「**Connection string**」→「URI」タブ
- [x] DATABASE_URL を取得（Render の環境変数に使用）

---

## STEP 3｜LINE Developers でボット作成 ✅

- [x] [developers.line.biz](https://developers.line.biz) → LINEアカウントでログイン
- [x] プロバイダー作成 → Messaging API チャネル作成
- [x] チャネルシークレット を取得
- [x] チャネルアクセストークン（長期）を発行・取得
- [x] 応答メッセージをオフに設定

> ⚠️ Webhook URL の設定は STEP 5 のデプロイ後に行います

---

## STEP 4｜Render でアプリをデプロイ ✅

- [x] [render.com](https://render.com) → GitHub アカウントでサインアップ
- [x] 「New +」→「Web Service」→ `ishi-kosei/task-remind` を選択
- [x] ビルド設定・環境変数（DATABASE_URL / LINE_CHANNEL_SECRET / LINE_CHANNEL_ACCESS_TOKEN）を設定
- [x] デプロイ完了・URL 取得

---

## STEP 5｜LINE Webhook URL を設定

- [ ] LINE Developers → チャネルページ →「**Messaging API設定**」タブ を開く
- [ ] 「**Webhook URL**」に以下を入力して「更新」

```
https://task-remind-xxxx.onrender.com/webhook
```

- [ ] 「**検証**」ボタンを押して「成功」と表示されることを確認
- [ ] 「**Webhookの利用**」を **オン** にする

---

## STEP 6｜LINE ボットを友だち追加して ID 登録

- [ ] LINE Developers →「Messaging API設定」タブ → **QRコード** をスマホで読み取る
- [ ] ボットを**友だち追加**する
- [ ] LINE でボットに何でもメッセージを送る（例:「こんにちは」）
- [ ] 「✅ LINE通知を設定しました！」と返信が来ることを確認

---

## STEP 7｜UptimeRobot でサーバーを常時起動

> Render 無料枠は15分で停止するため、5分おきにpingして起こし続ける設定

- [ ] [uptimerobot.com](https://uptimerobot.com) → 無料アカウント作成
- [ ] 「**Add New Monitor**」をクリック
  - Monitor Type: **HTTP(S)**
  - Friendly Name: `task-remind`
  - URL: `https://task-remind-xxxx.onrender.com/api/status`
  - Monitoring Interval: **5 minutes**
  - → 「Create Monitor」
- [ ] モニターが「**Up**」になっていることを確認

---

## STEP 8｜動作確認

- [ ] LINE でタスクを追加してみる
  ```
  明日 〇〇さんにお繋ぎ文送信
  ```
  → 「✅ タスクを追加しました！」と返ってくることを確認

- [ ] LINE で「**リスト**」と送る → 今日のタスク一覧が返ってくることを確認

- [ ] Web UI でタスクが表示されていることを確認
  ```
  https://task-remind-xxxx.onrender.com
  ```

- [ ] 翌朝 6:00 に LINE でタスク一覧通知が届くことを確認

## トラブルシューティング

| 症状 | 確認箇所 |
|------|---------|
| Render デプロイ失敗 | Render の「Logs」タブでエラーを確認 |
| LINE 検証エラー | Webhook URL の末尾に `/webhook` があるか確認 |
| LINE 通知が届かない | LINE でボットにメッセージを送って ID 登録したか確認 |
| DB 接続エラー | `DATABASE_URL` のパスワード部分が正しいか確認 |
| タスクが追加できない | LINE に「ヘルプ」と送って入力形式を確認 |
