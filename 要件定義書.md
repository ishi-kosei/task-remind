# タスクリマインダー アプリ 要件定義書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プロジェクト名 | TASK-Remind |
| 目的 | タスクの期限管理とリマインド通知の自動化 |
| 対象ユーザー | 個人利用（1名） |
| 作成日 | 2026-02-28 |

---

## 2. 背景・課題

- タスクの期限を忘れてしまうことがある
- LINEは日常的に使っており、通知を見落としにくい
- シンプルに「タスク内容と期限」を入力するだけで使えるツールがほしい

---

## 3. 機能要件

### 3-1. タスク管理（Web UI）

| # | 機能 | 詳細 |
|---|------|------|
| F-01 | タスク作成 | タイトル・期限日・期限時間（任意）を入力して登録 |
| F-02 | タスク一覧表示 | 未完了タスクを期限日順に表示。日付ごとにグループ化 |
| F-03 | 今日のタスク強調 | 当日期限のタスクは視覚的に強調表示 |
| F-04 | タスク完了 | チェックボタンで完了マーク（一覧から非表示） |
| F-05 | タスク削除 | 削除ボタンでタスクを削除 |

### 3-2. タスク追加（LINEチャット）

LINEボットにメッセージを送るだけでタスクを追加できる。**日付・時間だけ抜き取り、残りをそのままタイトルにする**シンプルな方式（外部AI不要・完全無料）。

| # | 機能 | 詳細 |
|---|------|------|
| L-01 | チャットからタスク追加 | 「日付 タイトル」の形式で送ると登録される |
| L-02 | 日付解析 | 今日・明日・明後日・M/D・M月D日 に対応 |
| L-03 | 時間解析 | HH:MM・HH時・HH時MM分・午後HH時・HH時半 に対応 |
| L-04 | タイトルはそのまま | 日付・時間を除いた残りテキストを加工せずタイトルにする |
| L-05 | 今日のタスク確認 | 「リスト」と送ると今日のタスク一覧を返信 |
| L-06 | ヘルプ | 「ヘルプ」と送ると使い方を返信 |
| L-07 | エラー返信 | 日付が読み取れない場合はエラーメッセージを返信 |

#### チャット入力例

| 送信メッセージ | 登録タイトル | 日付 | 時間 |
|----------------|-------------|------|------|
| `3/5 〇〇さんにお繋ぎ文送信` | 〇〇さんにお繋ぎ文送信 | 3/5 | — |
| `明日 〇〇さんお店探す` | 〇〇さんお店探す | 翌日 | — |
| `3月5日〜受取` | 〜受取 | 3/5 | — |
| `今日 14:00 打ち合わせ` | 打ち合わせ | 今日 | 14:00 |
| `3/10 午後3時 企画書提出` | 企画書提出 | 3/10 | 15:00 |

### 3-3. リマインド通知（LINE）

| # | 機能 | 詳細 |
|---|------|------|
| N-01 | 朝のリマインド | 毎朝 6:00（JST）に当日のタスク一覧をLINEで送信 |
| N-02 | 夕方のリマインド | 毎夕 18:00（JST）に当日のタスク一覧をLINEで送信 |
| N-03 | 直前リマインド | 時間指定があるタスクは、予定時刻の1時間前にLINE通知 |
| N-04 | LINE連携状態表示 | Web UI のヘッダーでLINE連携の有無を表示 |

### LINEメッセージ例

```
📋 今日のタスク - 2/28(土)
☀️ おはようございます！

・〇〇さんにお繋ぎ文送信
・〇〇さんお店探す
・打ち合わせ ⏰ 14:00

未完了: 3件
```

## 4. 非機能要件

| 項目 | 要件 |
|------|------|
| 可用性 | 24時間稼働（クラウドデプロイ） |
| スケール | 個人利用（同時接続1名） |
| 通知精度 | 1時間前通知は±1分以内 |
| タイムゾーン | 全処理を JST（Asia/Tokyo）で統一 |
| レスポンス | タスク追加・表示は1秒以内 |

## 5. 技術スタック

| 層 | 技術 |
|----|------|
| フロントエンド | HTML / CSS / Vanilla JS（SPA） |
| バックエンド | Node.js + Express |

## 6. データ設計

### tasks テーブル

| カラム | 型 | 説明 |
|--------|----|------|
| id | SERIAL PK | タスクID |
| title | TEXT | タスク内容 |
| due_date | DATE | 期限日 |
| due_time | TIME | 期限時間（任意） |
| completed | BOOLEAN | 完了フラグ |
| notified_before | BOOLEAN | 1時間前通知済みフラグ |
| created_at | TIMESTAMPTZ | 作成日時 |

### settings テーブル

| カラム | 型 | 説明 |
|--------|----|------|
| key | VARCHAR(100) PK | 設定キー |
| value | TEXT | 設定値 |

> `settings` に `line_user_id` を保存して通知先を管理する

## 8. LINE連携フロー

```
1. LINE Developers でボット作成
       ↓
2. Channel Secret / Access Token を取得
       ↓
3. Railway にデプロイ & 環境変数設定
       ↓
4. LINE Developers に Webhook URL を設定
   例: https://xxx.railway.app/webhook
       ↓
5. スマホでボットに友だち追加
       ↓
6. ボットにメッセージを送信
       ↓
7. 「✅ LINE通知を設定しました！」と返信
       ↓
8. 以降、6時・18時・1時間前に自動通知
```

## 9. 環境変数

| 変数名 | 必須 | 説明 |
|--------|------|------|
| DATABASE_URL | ✅ | PostgreSQL接続URL |
| LINE_CHANNEL_SECRET | ✅ | LINEチャネルシークレット |
| LINE_CHANNEL_ACCESS_TOKEN | ✅ | LINEチャネルアクセストークン |
| LINE_USER_ID | － | 通知先ユーザーID（省略可）|
| PORT | － | サーバーポート（デフォルト3000） |
